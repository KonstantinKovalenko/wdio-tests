# WDIO Tests (Trainee Automation Project)

## 📌 Overview & Goals
This is a trainee-level UI automation testing project built as part of a WebdriverIO course (YouTube series, 25 lessons).

The goal of this project is to learn and practice:
- Web UI test automation using WebdriverIO
- Basic test structure and framework setup
- Writing simple end-to-end test scenarios
- Using Page Object Model

This is my first automation testing project and serves as a foundation for more advanced QA projects.

---
## 🧪 What is tested
The project contains basic UI test cases such as:
- Page loading checks
- Element visibility validation
- Simple user interaction scenarios

Tests are written as introductory examples and focus on learning WDIO structure rather than production coverage.

---
## 🛠 Tech Stack
- WebdriverIO
- JavaScript
- Node.js
- Mocha (test runner)
- Chai / WDIO expect assertions

---
## 🚀 Install and Run tests

### Install dependencies

    npm install
    
### Run tests
    npx wdio run wdio.conf.js
    or
    npm run test

---
## 📁 Project Structure

    /test        - test specs  
    /pageobjects - page object models (if used)  
    /wdio.conf.js - WebdriverIO configuration
